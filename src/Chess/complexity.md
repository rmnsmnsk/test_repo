Оценка сложности:

Переборное решение
Всего перестановок n! вариантов (permutations генерирует все расстановки без повторений столбцов).
Для каждой перестановки вычисляется функция check_diagonal, которая считает сначала n, затем (n-1) .... 1.
Таким образом по формуле арифм прогрессии всего вычисляется n * (n - 1)/2 ~ O(n²).
Таким образом общая сложность - это O(n! * n²).

Рекурсивный способ
Неверные варианты отсекаются сразу, получается всего n! вариантов в худшем случае.
При этом для каждого шага функция safe проверяет только с уже поставленными ферзями, что требует O(row) операций.
В среднем это O(n) для каждого шага.
Таким образом получается O(n! * n).

Третье решение (итеративный)
Аналогично рекурсивному, в худшем случае перебирается n! вариантов.
Для каждой проверки используется функция is_valid, которая содержит две проверки check_column и check_diagonal.
Каждая из них имеет сложность O(k²), где k - длина текущей расстановки.
В худшем случае это O(n²) для полной расстановки.
Таким образом общая сложность O(n! * n²).
